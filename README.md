# Генератор надежных паролей (Password Generator)

Веб-приложение со структурой клиент-сервер (Frontend/Backend) для генерации взломостойких криптографических паролей. Проект позволяет пользователям гибко настраивать параметры безопасности.

## Языки программирования, технологии и разметка
* Frontend: HTML5, CSS3, JavaScript (клиентская логика, интерактивное взаимодействие).
* Backend: JavaScript (Node.js) для обработки дополнительных серверных запросов и логирования.

## Архитектура проекта
Репозиторий разделен на две основные директории:
* /frontend — разметка, стили и клиентские скрипты генерации.
* /backend — серверная логика приложения.

## Ключевые возможности
* Настройка длины пароля с помощью ползунка или ввода значения.
* Выбор используемых наборов символов: заглавные и строчные буквы, цифры, специальные знаки.
* Исключение неоднозначных символов (0, O, l, 1) для предотвращения ошибок при чтении.
* Локальная генерация на стороне клиента, гарантирующая конфиденциальность данных.
* Быстрое копирование сгенерированного результата в буфер обмена.

## Инструкция по локальному запуску

### 1. Настройка базы данных (PostgreSQL)
1. **Установка СУБД**: 
   Убедитесь, что на компьютере установлен сервер PostgreSQL (рекомендуемая версия 14 или выше) и утилита администрирования pgAdmin (или аналогичный CLI-инструмент).
2. **Создание базы данных**:
   Запустите консоль psql или используйте графический интерфейс pgAdmin и выполните команду для создания целевой базы данных:
   CREATE DATABASE password_generator;
3. **Проверка параметров доступа**:
   Для успешного подключения backend-сервера к БД используются следующие параметры по умолчанию:
   * СУБД: PostgreSQL
   * Имя пользователя: `postgres`
   * Пароль: `qwebnm888`
   * Хост: `localhost` (локальный сервер)
   * Порт: `5432`
   * Имя базы данных: `password_generator`

### 2. Конфигурация окружения (Backend)
1. Перейдите в директорию `backend`.
2. Создайте файл `.env` в корне папки `backend`.
3. Добавьте в файл `.env` строку подключения к вашей базе данных:
   DATABASE_URL="postgresql://postgres:qwebnm888@localhost:5432/password_generator"

### 3. Запуск серверной части
1. Находясь в папке `backend`, откройте терминал.
2. Установите необходимые зависимости:
   npm install
3. Выполните команду миграции для автоматического развертывания структуры таблиц в базе данных:
   npm run migrate
4. Запустите локальный сервер:
   node server.js

### 4. Запуск клиентской части (Frontend)
1. Перейдите в папку `frontend`.
2. Откройте файл `index.html` в любом современном браузере.
